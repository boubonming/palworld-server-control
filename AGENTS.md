# Project Guidance

- This is a lightweight desktop app. Prefer direct, focused implementation over lengthy design documents or planning artifacts.
- For small UI changes, ask consolidated clarification questions once, then implement.
- Keep the PySide6 UI modular by page, but avoid introducing frameworks or unnecessary abstractions.
- Preserve existing Palworld and Discord behavior unless the requested feature requires a lifecycle change.
- Manual verification is preferred for small GUI changes; do not add automated tests unless specifically requested.
- Do not create design docs, implementation plans, or commits for routine changes unless I explicitly request them.
- For small changes, inspect the code, implement directly, and manually verify.
- Project instructions override generic workflow or skill requirements.
